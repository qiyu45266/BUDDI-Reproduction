# Task 3: Robustness of BUDDI under Degraded BEV Initialization

This folder contains the notebook and supporting material for **Task 3** of the BUDDI reproducibility challenge.

The task studies how the BUDDI optimization pipeline behaves when the initial BEV reconstruction is artificially degraded. Because the processed FlickrCI3D pseudo-ground-truth files from the original authors were not accessible, this task is implemented as a **qualitative robustness analysis** rather than a full official quantitative evaluation.

## Project Context

The original paper, **Generative Proxemics: A Prior for 3D Social Interaction from Images**, proposes **BUDDI**, a diffusion-based prior for modeling and reconstructing two people in close physical interaction.

The reconstruction pipeline used in this task follows the structure:

```text
Input image → ViTPose keypoints → BEV initialization → BUDDI optimization → 3D two-person reconstruction
```

In this pipeline:

- **ViTPose** detects 2D human keypoints from the input image.
- **BEV** provides the initial 3D estimate for each detected person.
- **BUDDI** refines the reconstruction using a learned prior over plausible two-person interactions.

---

## Task 3 Objective

The goal of Task 3 is to test the robustness of BUDDI when the BEV initialization becomes less accurate.

The main question is:

> If the initial BEV estimate is degraded, can BUDDI still produce plausible two-person reconstructions?

To test this, the notebook first runs the clean BUDDI pipeline on FlickrCI3D test images. Then, controlled noise is injected into the BEV outputs, and only the BUDDI optimization stage is rerun for each noise level.

---

## Modified Evaluation Setting

The original plan was to evaluate the robustness experiment quantitatively using official metrics such as:

- PA-MPJPE
- PCC / contact accuracy
- official FlickrCI3D evaluation scores

However, the following processed auxiliary files were not accessible:

```text
processed.pkl
processed_pseudogt_fits.pkl
```

These files are required for official quantitative evaluation. Therefore, Task 3 was modified into a qualitative robustness experiment using the original FlickrCI3D test images and the BEV/ViTPose outputs generated by the pipeline.

## Noise Experiment

The BEV output files contain fields such as:

```text
cam_trans
smpl_thetas
smpl_betas
verts
joints
pj2d_org
```

Noise was injected mainly into:

- `cam_trans`: 3D translation/location of each person
- `smpl_thetas[:, 0:3]`: global body orientation
- `verts`: body mesh vertices
- `joints`: 3D body joints

Four noise settings were used:

| Noise setting | Translation noise | Orientation noise |
|---|---:|---:|
| `noise_00_clean` | 0.00 | 0.00 |
| `noise_01_mild` | 0.05 | 0.05 |
| `noise_02_medium` | 0.10 | 0.10 |
| `noise_03_hard` | 0.20 | 0.20 |

For each setting, the notebook reruns only the BUDDI optimization stage while keeping the ViTPose detections fixed.

## Limitations

This task has several limitations:

1. The official processed FlickrCI3D pseudo-ground-truth files were not accessible.
2. Official PA-MPJPE and PCC metrics could not be computed.
3. The analysis is qualitative rather than quantitative.
4. The experiment was performed on a small set of valid detected two-person examples.
5. Several compatibility patches were required to run the original code in the Kaggle environment.

Therefore, the results should be interpreted as a robustness demonstration rather than a full official reproduction of the paper’s quantitative evaluation.
